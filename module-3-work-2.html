Assignment brief

This unit focused on regular expressions and recursion in software solutions, and their affect on system security. I was asked to provide answers to the following questions:
1. What is ReDOS and what part do ‘Evil Regex’ play?
2. What are the common problems associated with the use of regex? How can these be mitigated?
3. How and why could regex be used as part of a security solution?

Work submission

Regular expressions allow a way to search through and process strings, and allow for validating and processing of that data. They are used by a number of different frameworks, such as being .NET's default validation method. Regular expression Denial of Service (ReDoS) is a form of DoS attack which specifically targets regular expressions by exploiting the fact that input size has an exponential impact on the performance of a query. More complex expressions can be maliciously crafted to paralyse applications. Evil regex refers to the specific regular expressions that are crafted to cause a ReDoS attack. They are defined by OWASP as containing groupings with repetition, and additional repetition and alternation with overlapping inside of the groups

Regex can be a powerful method of searching and processing large amounts of data, however it is not without its weaknesses. These weaknesses include:
- Expressions are compiled at runtime, meaning it cannot provide feedback on potential errors.
- Performance can be impacted when a query has to be processed in quadratic time (this is what makes ReDoS attacks possible).
- They can be vulnerable to injection attacks.
These weaknesses can be mitigated by the following:
- Using specific libraries and tools which are designed to handle regex securely.
- Implementing code to sanitise and validate inputs.
- Ensuring expressions are carefully crafted and appropriately tested before being pushed live.

Security solutions rely on being able to reliably spot an unwanted connection or injection, as well as being able to reliably find relevant information within system logs. Both of these use cases are very suitable for regex. Regex can be harnessed to validate inputs entered by users, and can powerfully determine what inputs are suitable based on a single expression. They can also be used to specify firewall rules by blocking connections based on key expressions.
